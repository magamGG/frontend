## 1. API 구조 개선

### 1.1 문제점: 직접 API 호출

**❌ Before (문제가 있는 코드)**

```javascript
// 각 페이지에서 직접 axios 호출
import axios from 'axios';

const handleLogin = async () => {
  try {
    const response = await axios.post('http://localhost:8888/api/auth/login', {
      email: emailInput,
      password: passwordInput
    });
    // 성공 처리
  } catch (error) {
    // 에러 처리
  }
};
```

**문제점:**
- API 엔드포인트가 하드코딩되어 있음
- 각 페이지마다 중복된 에러 처리 코드
- API URL 변경 시 여러 파일 수정 필요
- 테스트하기 어려움

### 1.2 해결책: 서비스 레이어 패턴

**✅ After (개선된 코드)**

```javascript
// src/api/config.js - API 엔드포인트 중앙 관리
export const API_ENDPOINTS = {
  AUTH: {
    LOGIN: `/api/auth/login`,
    LOGOUT: `/api/auth/logout`,
    // ...
  },
};

// src/api/services.js - 서비스 객체로 추상화
export const authService = {
  login: (credentials) => {
    return api.post(API_ENDPOINTS.AUTH.LOGIN, credentials);
  },
};

// 페이지에서 사용
import { authService } from '@/api';

const handleLogin = async () => {
  try {
    await authService.login({ email: emailInput, password: passwordInput });
    toast.success('로그인 성공');
  } catch (error) {
    toast.error(error?.message || '로그인 실패');
  }
};
```

**장점:**
- ✅ API 엔드포인트 중앙 관리 (`config.js`)
- ✅ 비즈니스 로직과 API 호출 분리
- ✅ 재사용 가능한 서비스 함수
- ✅ 테스트 용이성 향상
- ✅ 유지보수성 향상

### 1.3 서비스 레이어 구조

```
src/api/
├── axios.js          # Axios 인스턴스 설정 (인터셉터, 기본 헤더 등)
├── config.js         # API 엔드포인트 정의
└── services.js       # 도메인별 서비스 객체
```

**axios.js 예시:**

```javascript
import axios from 'axios';
import { API_BASE_URL, API_TIMEOUT } from './config';

const api = axios.create({
  baseURL: API_BASE_URL,
  timeout: API_TIMEOUT,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 요청 인터셉터: 토큰 자동 추가
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 응답 인터셉터: 에러 통일 처리
api.interceptors.response.use(
  (response) => response.data,
  (error) => {
    // 에러 처리 로직
    return Promise.reject(error);
  }
);

export default api;
```


## 3. 에러 처리 패턴

### 3.1 통일된 에러 처리

**❌ Before: 각 페이지마다 다른 에러 처리**

```javascript
// 페이지 A
try {
  await axios.post('/api/login', data);
} catch (error) {
  alert('로그인 실패');
}

// 페이지 B
try {
  await axios.post('/api/signup', data);
} catch (error) {
  console.error(error);
  alert(error.message);
}
```

**✅ After: 통일된 에러 처리**

```javascript
// 서비스 레이어에서 에러 처리
import { toast } from 'sonner';

const handleLogin = async () => {
  setIsLoading(true);
  try {
    await authService.login({ email: emailInput, password: passwordInput });
    toast.success('로그인 성공');
    // 성공 후 처리
  } catch (error) {
    const errorMessage = error?.message || '로그인에 실패했습니다. 다시 시도해주세요.';
    toast.error(errorMessage);
    console.error('Login error:', error);
  } finally {
    setIsLoading(false);
  }
};
```

### 3.2 에러 처리 패턴

```javascript
try {
  // API 호출
  await service.someMethod(data);
  
  // 성공 처리
  toast.success('작업이 완료되었습니다.');
  
} catch (error) {
  // 1. 사용자 친화적 메시지
  const errorMessage = error?.response?.data?.message 
    || error?.message 
    || '작업에 실패했습니다. 다시 시도해주세요.';
  
  // 2. 사용자에게 알림
  toast.error(errorMessage);
  
  // 3. 개발자용 로그 (콘솔)
  console.error('Operation error:', error);
  
  // 4. 필요시 추가 처리 (예: 특정 에러 타입별 처리)
  if (error?.response?.status === 401) {
    // 인증 에러 처리
  }
} finally {
  // 5. 항상 실행되는 정리 작업 (로딩 상태 해제 등)
  setIsLoading(false);
}
```

### 3.3 Axios 인터셉터를 통한 전역 에러 처리

```javascript
// src/api/axios.js
api.interceptors.response.use(
  (response) => response.data,
  (error) => {
    // 401: 인증 실패 → 로그인 페이지로 리다이렉트
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    
    // 403: 권한 없음
    if (error.response?.status === 403) {
      toast.error('접근 권한이 없습니다.');
    }
    
    // 500: 서버 에러
    if (error.response?.status >= 500) {
      toast.error('서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요.');
    }
    
    return Promise.reject(error);
  }
);
```

## 4. 로딩 상태 관리

### 4.1 로딩 상태 패턴

**❌ Before: 로딩 상태 없음**

```javascript
const handleSubmit = async () => {
  await api.post('/api/data', data);  // 사용자가 버튼을 여러 번 클릭 가능
};
```

**✅ After: 로딩 상태 관리**

```javascript
const [isLoading, setIsLoading] = useState(false);

const handleSubmit = async () => {
  setIsLoading(true);
  try {
    await service.createData(data);
    toast.success('저장되었습니다.');
  } catch (error) {
    toast.error('저장에 실패했습니다.');
  } finally {
    setIsLoading(false);  // 항상 로딩 상태 해제
  }
};

// UI에서 사용
<Button disabled={isLoading}>
  {isLoading ? '저장 중...' : '저장'}
</Button>
```


**문제점:**

2. 에러 처리 부족
3. 로딩 상태 없음
4. 변수명이 모호함 (`formData`)
5. 사용자 피드백 부족 (alert 사용)

#### After: 개선된 코드

```javascript
// SignupPage.jsx (After)
import { useState } from 'react';
import { toast } from 'sonner';
import { memberService } from '@/api';

export function SignupPage() {
  const [isLoading, setIsLoading] = useState(false);
  const [signupFormData, setSignupFormData] = useState({
    name: '',
    email: '',
    password: '',
    confirmPassword: '',
  });

  const handleSubmit = async (e) => {
    e.preventDefault();
    
    // 유효성 검사
    if (signupFormData.password !== signupFormData.confirmPassword) {
      toast.error('비밀번호가 일치하지 않습니다.');
      return;
    }

    setIsLoading(true);
    try {
      await memberService.register(signupFormData);
      toast.success('회원가입이 완료되었습니다.');
      // 성공 후 처리
    } catch (error) {
      const errorMessage = error?.message || '회원가입에 실패했습니다. 다시 시도해주세요.';
      toast.error(errorMessage);
      console.error('Signup error:', error);
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={signupFormData.email}
        onChange={(e) => setSignupFormData({ ...signupFormData, email: e.target.value })}
        disabled={isLoading}
        placeholder="이메일"
      />
      <input
        type="password"
        value={signupFormData.password}
        onChange={(e) => setSignupFormData({ ...signupFormData, password: e.target.value })}
        disabled={isLoading}
        placeholder="비밀번호"
      />
      <button type="submit" disabled={isLoading}>
        {isLoading ? '가입 중...' : '회원가입'}
      </button>
    </form>
  );
}
```

**개선 사항:**
1. ✅ 서비스 레이어 사용 (`memberService.register`)
2. ✅ 로딩 상태 관리 (`isLoading`)
3. ✅ 통일된 에러 처리 (`toast.error`)
4. ✅ 명확한 변수명 (`signupFormData`)
5. ✅ 사용자 피드백 개선 (`toast` 사용)
6. ✅ 유효성 검사 추가
7. ✅ 로딩 중 입력/버튼 비활성화

```javascript
    toast.success('인증 코드가 이메일로 전송되었습니다.');
    setStep(PASSWORD_RESET_STEPS.VERIFY);
  } catch (error) {
    const errorMessage = error?.message || '이메일 전송에 실패했습니다.';
    toast.error(errorMessage);
  } finally {
    setIsLoading(false);
  }
};

const handleVerifyCode = async (e) => {
  e.preventDefault();
  if (!verificationCodeInput) {
    toast.error('인증 코드를 입력해주세요.');
    return;
  }

  setIsLoading(true);
  try {
    await authService.verifyCode(emailInput, verificationCodeInput);
    toast.success('인증 코드가 확인되었습니다.');
    setStep(PASSWORD_RESET_STEPS.RESET);
  } catch (error) {
    const errorMessage = error?.message || '인증 코드 확인에 실패했습니다.';
    toast.error(errorMessage);
  } finally {
    setIsLoading(false);
  }
};
```
