주간 업무 계획 및 세부 진행 내용

1. 주간 업무 계획 (요약) — 구현 기준

프론트
- API 엔드포인트 설정(config.js): 인증, 회원(에이전시별·담당자별·배정), 출석·근태(시작/종료/오늘상태/이력/통계), 연차(신청/목록/잔액/승인·반려), 프로젝트·칸반, 캘린더, 알림, 건강 설문·일일체크, 에이전시(가입요청/승인·거절) — 구현됨.
- API 요청 로직(services.js): authService, memberService, attendanceService, leaveService, projectService, calendarService, notificationService, healthService, agencyService 연동 — 구현됨.
- 페이지 구성: admin(대시보드, 팀, 캘린더, 건강, 근태, 마이페이지, 워케이션, 프로젝트), agency(승인, 배정, 팀, 가입요청, 연차설정, 프로젝트, 대시보드, 건강, 마이페이지, 워케이션), artist(대시보드, 캘린더, 건강, 팀, 워케이션, 프로젝트), health(설문·모니터링·위험분석·미검자 등), 공통(로그인, 회원가입, 마이페이지, 캘린더, 출석) — 구현됨.
- UI: shadcn/ui 기반 컴포넌트, 모달(HealthCheckModal, LeaveRequestModal, UniversalHiatusModal, ProjectListModal 등), 레이아웃·헤더·사이드바 — 구현됨.

ERD
- 로직에 맞는 테이블 재설계 및 수정 — schema.sql 반영됨.
- 기존: AGENCY, MEMBER, PROJECT, HEALTH_SURVEY, CALENDAR_EVENT, NOTIFICATION, LEAVE_BALANCE, ATTENDANCE, ATTENDANCE_REQUEST, LEAVE_HISTORY 등 — 구현됨.
- 추가·연동: MANAGER, ARTIST_ASSIGNMENT(작가–담당자 배정), NEW_REQUEST(에이전시 가입 요청), DAILY_HEALTH_CHECK(일일 건강 체크), HEALTH_SURVEY_QUESTION·HEALTH_SURVEY_RESPONSE 등 — 구현됨.

백엔드
- 인증: AuthController에서 AuthService.login(요청) 호출로 로그인·토큰(회원정보) 반환 처리함.
- 회원·직원: MemberController에서 MemberServiceImpl로 회원가입·에이전시별 회원/담당자/작가 조회·상세·배정·해제·삭제·에이전시 제외·마이페이지·프로필·이미지·직원 통계 처리. ManagerController에서 ArtistAssignmentService로 작가–담당자 배정/해제 처리함.
- 출근: AttendanceController에서 AttendanceServiceImpl로 출근 시작(DailyHealthCheckService에 일일 건강 체크 저장 후 ATTENDANCE에 출근 기록), 출근 종료(ATTENDANCE에 퇴근 기록), 오늘 상태(ATTENDANCE 마지막 타입 조회), 이력·통계(ATTENDANCE 기준 집계) 처리함.
- 근태(휴가·연차): AttendanceRequestController(/api/leave)에서 AttendanceService로 근태 신청 생성(ATTENDANCE_REQUEST 저장·담당자 알림), 내 신청·에이전시별·대기 목록 조회, 현재 적용 상태(승인된 신청 중 오늘 포함 여부), 승인/반려(상태 변경·신청자 알림) 처리함.
- 에이전시: AgencyController에서 AgencyServiceImpl로 가입 요청(NEW_REQUEST 저장·담당자 알림), 요청 목록 조회, 승인(NEW_REQUEST·MEMBER.AGENCY_NO 갱신·LEAVE_BALANCE 초기화·담당자면 MANAGER 등록)·거절 처리함.
- 건강: HealthSurveyController에서 HealthSurveyService로 설문 질문 조회·응답 제출(HEALTH_SURVEY_RESPONSE 등) 처리. 일일 건강 체크는 출근 시작 시 AttendanceServiceImpl에서 DailyHealthCheckService.createDailyHealthCheck 호출로 DAILY_HEALTH_CHECK 저장함.
- 알림: NotificationController에서 NotificationService로 회원별 알림 목록·읽음·전부 읽음·삭제 처리함.
- 캘린더: CalendarEventService만 존재, Controller 없음 — API 미구현. 프로젝트·칸반: ProjectService·KanbanBoardService만 존재, Controller 없음 — API 미구현.


2. 공통 작업

폼 통일: UniversalFormModal 컴포넌트로 휴재/워케이션/연차/야근/마감연장 등 신청 폼 통합, formType으로 구분

CRUD: 작품·팀·근태 목록 등 Create/Read/Update/Delete 구현, API 연동 및 폼 검증

버튼 동작: 모든 버튼 onClick 연결, 모달 열기/닫기, 페이지 라우팅, 요구사항 문서 기준 점검


3. 디자인 및 UI/UX

작가 대시보드: 상태별 화면(일반/워케이션/단기·시즌·장기 휴재), Todo 리스트, 오늘 할 일·D-day·진행 현황, 차트·그래프

캘린더: 근태 유형별 색상(연차/휴재/워케이션/야근), 휴재 일수 표시, 작품별 필터

알림·설정: 알림 컴포넌트, 설정 페이지, 알림 설정 UI


4. ERD 설계

근태: ATTENDANCE_TYPE, ATTENDANCE_REQUEST, HIATUS_DETAIL, WORKCATION_DETAIL, 연재 일정, 워케이션 일일 출퇴근·업무 보고, 연차 등

건강: HEALTH_SURVEY, DAILY_HEALTH_SURVEY, HEALTH_REPORT, 건강 점수·위험도 기준

스케줄: 작품별 연재 주기, 마감일, 에피소드 진행률, TASK, TODO, 팀 Todo


5. 기능 구현

체크인/아웃: 로그인 시 출근 여부 확인 → 건강 설문 → 오늘 할 일 보고, 퇴근 시 미완료 계획 폼, 11시간 경과 자동 퇴근, 야근 신청

워케이션: 전용 대시보드, 시작/종료 보고서, 일일 업무 보고, 위치 관리, 미보고 알림

휴재: 휴재 신청·승인, 장기/시즌 휴재 관리, 휴재 중 업무 배정, 휴재 일수 계산·표시


6. 피드백 반영

명칭: 프로젝트 관리 → 작품 관리, 휴가 관리 → 근태 관리, 긴급 프로젝트 → 진행 프로젝트

페이지: 작가 마이페이지 진행작품 섹션 조정, 스케줄 종일 체크·이번주 요약 제거, 전사 건강 → 개별 건강 상태 확인

팀 관리: 팀 그룹별 역할(작가/어시/보조), 출근 상태 필터, 프로필·이메일/메시지 연동


7. 세부 진행 내용 (핵심만)

폼·모달: UniversalHiatusModal, LeaveRequestModal, HealthCheckModal 별도 구현. 캘린더·레이아웃에서 연차/근태 모달 연동됨.

CRUD·API 연동: 팀(담당자·작가 조회·상세·memberService), 마이페이지(프로필·이미지·삭제·통계), 대시보드(출근 시작/종료·오늘 상태·근태 상태·attendanceService·leaveService) 등 페이지별로 호출 구현됨.

버튼·라우팅: react-router-dom 라우팅, 모달 열기/닫기, 대시보드·팀·캘린더·마이페이지 등 이동 연결됨.

대시보드·캘린더·알림: leaveService.getCurrentStatus·attendanceService.getTodayStatus로 근태 상태 표시. recharts·chart 컴포넌트 사용. Header 알림·notificationService 연동됨.
