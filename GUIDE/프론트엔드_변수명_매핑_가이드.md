## 1. 변수명 변환 규칙

### 기본 규칙
- **DB 스키마**: `SNAKE_CASE` (대문자 + 언더스코어)
- **프론트엔드**: `camelCase` (소문자 시작 + 카멜케이스)

### 변환 예시
```
MEMBER_NO → memberNo
MEMBER_NAME → memberName
MEMBER_CREATED_AT → memberCreatedAt
ATTENDANCE_REQUEST_TYPE → attendanceRequestType
```

### 특수 케이스
- `_NO` 접미사 → `No` (예: `MEMBER_NO` → `memberNo`)
- `_AT` 접미사 → `At` (예: `CREATED_AT` → `createdAt`)
- `_URL` 접미사 → `Url` (예: `FILE_URL` → `fileUrl`)
- `_ID` 접미사 → `Id` (예: `MEMBER_ID` → `memberId`) - ⚠️ 단, MEMBER 테이블에는 `MEMBER_ID` 필드가 없고 `MEMBER_EMAIL`을 사용

## 2. 엔티티별 변수명 매핑

### 2.1 AGENCY (에이전시)

```javascript
// DB 필드 → 프론트엔드 변수
AGENCY_NO → agencyNo
AGENCY_NAME → agencyName
AGENCY_CODE → agencyCode
AGENCY_LEAVE → agencyLeave  // 기본 연차 일수 (기본값: 15)
```

---

### 2.2 MEMBER (회원)

⚠️ **주의**: DB에는 `MEMBER_ID` 필드가 없습니다. 로그인 시 `MEMBER_EMAIL`을 사용합니다.

```javascript
// DB 필드 → 프론트엔드 변수
MEMBER_NO → memberNo
AGENCY_NO → agencyNo  // NULL 허용
MEMBER_NAME → memberName
MEMBER_PASSWORD → memberPassword
MEMBER_EMAIL → memberEmail  // UNIQUE, 로그인 ID로 사용
MEMBER_PHONE → memberPhone
MEMBER_STATUS → memberStatus  // 'ACTIVE' | 'BLOCKED'
MEMBER_PROFILE_IMAGE → memberProfileImage
MEMBER_PROFILE_BANNER_IMAGE → memberProfileBannerImage
MEMBER_ROLE → memberRole  // '웹툰 작가' | '웹소설 작가' | '어시스트 - 채색' | '어시스트 - 조명' | '어시스트 - 배경' | '어시스트 - 선화' | '어시스트- 기타' | '에이전시 관리자' | '담당자'
MEMBER_CREATED_AT → memberCreatedAt
MEMBER_UPDATED_AT → memberUpdatedAt
```

## 4. 주의사항

### 4.1 MEMBER 테이블
- **`MEMBER_ID` 필드 없음**: DB에는 `MEMBER_ID` 필드가 없고, `MEMBER_EMAIL`이 UNIQUE로 사용됩니다.
- **로그인**: `memberId`로 이메일을 사용해야 합니다.
- **`AGENCY_NO`**: NULL 허용 (에이전시에 속하지 않은 회원 가능)

### 4.2 날짜/시간 타입
- **ATTENDANCE_REQUEST**: `ATTENDANCE_REQUEST_START_DATE`, `ATTENDANCE_REQUEST_END_DATE`는 DATETIME 타입입니다.
- **KANBAN_CARD**: `KANBAN_CARD_STARTED_AT`, `KANBAN_CARD_ENDED_AT`는 DATE 타입입니다.

### 4.4 날짜 형식
- **DB**: `DATE` (YYYY-MM-DD), `DATETIME` (YYYY-MM-DD HH:mm:ss)
- **프론트엔드**: ISO 8601 문자열 권장 (예: `"2026-01-26T10:00:00"`)

### 4.5 상태 값
- **DB**: `'Y'/'N'` 문자열 또는 `'ACTIVE'/'BLOCKED'` 등
- **프론트엔드**: 
  - boolean으로 변환 가능한 경우: `status === 'Y'` → `true`
  - 또는 그대로 문자열 사용: `notificationStatus === 'Y'`

### 4.6 외래키
- 외래키 필드는 프론트엔드에서도 동일한 변수명 사용
- 예: `MEMBER_NO` → `memberNo`, `PROJECT_NO` → `projectNo`

### 4.7 배열/리스트 변수
- 프론트엔드에서는 `List` 접미사 사용 권장
- 예: `memberList`, `projectList`, `leaveRequestList`

### 4.8 API 서비스 사용
- 모든 API 호출은 `src/api/services.js`의 서비스 객체를 통해 수행
- 직접 `axios` 또는 `fetch` 호출 금지
- 예: `import { leaveService } from '@/api';`
